The user requested a web application in Romanian to manage attendance for a club or organization.

**PRODUCT REQUIREMENTS:**
1.  **Authentication:** Users must log in. A default  user with the password  is required. The registration page should be removed.
2.  **Dashboard:** After login, the user sees a dashboard displaying the current date.
3.  **Two Tables on One Page:** The dashboard should display two tables sequentially: Membri (Members) and Invitați (Guests).
4.  **Membri Table:**
    *   Columns: Nr. (re-numbered on sort), Prenume, Nume, Nume Înlocuitor (Substitute Name), Prezent (Checkbox), Taxă (Fee), Total Lună (Total for the month).
    *   The Nr. column should be sequential and update after sorting.
    *   Members should be sorted alphabetically by Prenume then Nume.
    *   A grand total row for Prezent, Taxă, and Total Lună is required.
    *   The Total Lună for a member should update when a new daily Taxă is entered.
5.  **Invitați Table:**
    *   A form is present to add new guests.
    *   Columns: Nr., Prenume, Nume, Companie, Invitat de, Taxă, Înlocuitor (checkbox).
    *   Guest data should be non-editable (plain text) after being added.
6.  **Substitute Logic:**
    *   When a Nume Înlocuitor is entered for a member, a new entry is automatically created in the Invitați table.
    *   This new guest entry is marked as a substitute (checkbox checked and disabled), the Invitat de field is automatically populated with the member's name, and the Companie is set to Înlocuitor.
7.  **Data Persistence:** All data (attendance, fees, guests) must be saved in the database.
8.  **Attendance History:** The user can select a date from a calendar to view historical attendance data. Dates with saved records should be marked with a blue hollow circle in the calendar.
9.  **Member Management:** A separate page (Administrare Membri) allows adding, editing, and deleting members. The list should be sorted alphabetically.
10. **Settings:** A page to change the admin password.
11. **Layout & Styling:**
    *   A collapsible side navigation menu.
    *   Custom column widths for better readability (Nr., Taxa, Înlocuitor narrower; Prenume, Nume wider).
    *   PDF export should be triggered by a button and use  to capture only the main content area with minimal margins.
